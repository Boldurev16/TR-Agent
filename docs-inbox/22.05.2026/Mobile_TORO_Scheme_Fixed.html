Процесс создания приложения «Мобильное ТОРО»

Согласно корпоративным правилам и анализу документов IT.R. 24-12 / IT.P. 24-11

graph TD
    %% Узлы с английскими ID для избежания ошибок парсера
    Start[Идея: Создание Мобильного ТОРО] --> Step1{Шаг 1:
Обращение к
Корп. Архитектору}
    
    Step1 -->|Утверждение| Step2[Шаг 2:
Анализ Деревьев мобильности]
    
    Step2 --> Choice{Выбор стратегии}
    
    %% Путь Вендора
    Choice -->|Покупка| PathBuy[Путь Б: Выбор вендора]
    PathBuy --> V1[Обзор рынка и реестра ПО РФ]
    V1 --> V2[Формирование списка вендоров]
    V2 --> V3[Оценка зрелости продукта
(как в IT.P. 24-11)]
    V3 --> V4[Пилотное внедрение]

    %% Путь Разработки
    Choice -->|Разработка| PathBuild[Путь А: Внутренняя разработка]
    PathBuild --> I1[Формирование ТЗ]
    I1 --> I2[Выбор стега по стандартам
Деревьев мобильности]
    I2 --> I3[Создание MVP/Прототипа]
    I3 --> I4[Тестирование релевантности]

    %% Валидация
    V4 --> Validation{Шаг 3:
Валидация решения}
    I4 --> Validation
    
    Validation -->|Успех| Step4[Шаг 4:
Интеграция в архитектуру]
    Validation -->|Отказ| BackToStart[Возврат к анализу
или отказ от проекта]
    
    Step4 --> EndNode[Шаг 5:
Масштабирование]
