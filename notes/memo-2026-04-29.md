# Мемо: прогресс 29.04.2026

Краткая фиксация по проекту **TR Agent** (форк «10 001 PDF»): локальный PDF + AnythingLLM + LM Studio. Гайд: [AiManual](https://ai-manual.ru/article/avtonomnyij-issledovatelskij-agent-na-anythingllm-kak-obrabotat-10-000-pdf-lokalno-na-32-gb-ozu/).

## Что сделано сегодня

- **Стек:** Docker (`local-ai-workspace-anythingllm`, образ `mintplexlabs/anythingllm:latest`), AnythingLLM **v1.12.1**, LM Studio на Windows, чат через **LM Studio** с `host.docker.internal` из контейнера.
- **Воркспейс:** `tr-agent` (в интерфейсе — TR Agent), промпт исследовательского агента и низкая температура согласованы с идеей гайда.
- **Диагностика:** освоена команда просмотра логов контейнера:
  ```powershell
  docker logs local-ai-workspace-anythingllm --tail 200
  ```
  (используется **имя контейнера**, не имя образа.)
- **Скрипты:** исправлен отступ в `scripts/make_pdf_batches.py` (`IndentationError` у `continue`); батчи для PDF из `docs-inbox` → `docs-batches` работают.
- **Эмбеддинги:** выяснено, что настройки встраивания находятся в **Настройки экземпляра → Поставщики ИИ → Настройки встраивания**; вкладка воркспейса «Векторная база данных» задаёт пороги и счётчик векторов, но не выбор модели эмбеддингов.
- **Причина `failed` у PDF (по логам):** после разборки файла падала **векторизация**: LM Studio отвечал `400 "No models loaded..."` и/или для эмбеддингов ошибочно указывалась **чат-модель** (Gemma). Решение: для эмбеддингов — отдельная модель (**например** `text-embedding-nomic-embed-text-v1.5`), в LM Studio должен быть включён **Local Server** и **загружена** соответствующая модель в момент индексации.
- **«Мало векторов» (например, 2):** это часто норма: **вектор = чанк**, не страница. При большом **chunk size** (в логах встречалось **8192**) один PDF может дать **один** чанк; два файла → два вектора. Для более мелкого поиска — уменьшить размер чанка в настройках экземпляра (**Text Splitting and Chunking**), затем сбросить векторы воркспейса и переиндексировать.
- **Nemotron и пустой ответ в чате:** у части «thinking»-моделей ответ уходит в `reasoning_content`, а UI AnythingLLM может показывать только `content`; помогает отключение thinking в LM Studio или другая модель для проверки.
- **Modelfile (`config/Modelfile`):** относится только к **Ollama** (`ollama create ...`), не управляет LM Studio и не подменяет модель в AnythingLLM. Какую модель запрашивает инференс, задаёт **настройка «Модель чата» воркспейса** в AnythingLLM (например `google/gemma-4-e4b`).

## Риски / ограничения

- Встроенный **`all-MiniLM-L6-v2`** ориентирован на английский; для русскоязычных документов предпочтительнее **мультиязычный** эмбеддер (например nomic через LM Studio).
- PDF без текстового слоя требуют OCR (в логах AnythingLLM уже фигурировал OCR для `Презентация1.pdf`).
